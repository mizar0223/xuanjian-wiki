## 二、总体策略：先筛后审

```
153 个未知页面
    ↓ 阶段0：预筛选（脚本自动化）
    ├─ A类：有原文直接证据 → 批量修正
    ├─ B类：有间接推断依据 → Agent 分批审计
    ├─ C类：天然无道统属性 → 批量标注说明
    └─ D类：确实无法判断 → 维持现状
```

### 筛选标准

| 分类 | 判定条件 | 处理方式 | 预计数量 |
|------|----------|----------|----------|
| **A类** | wiki 引用区或 context 中有「是XX道统」「属XX」等直接文字 | 人工确认后脚本批量修正 | 15-25 |
| **B类** | 有持有者道统、功能/材料描述、名称关键词等间接证据 | 3 Agent 并行审计（v3.1 流程） | 40-60 |
| **C类** | 灵物/灵资/材料类，全文无任何道统相关描述 | 批量标注「天然属性，无道统归属」 | 60-80 |
| **D类** | 无直接证据、无可靠间接依据 | 维持「未知 ❓」 | 10-20 |

## 三、分批执行计划

### 阶段 0：预筛选（脚本自动化，1 轮）

**目标**：产出分类清单

**脚本任务**：
1. 遍历 153 个未知页面
2. 读取各页 wiki 的【原文引用】区
3. 读取宽窗 `context/{品类}/{实体}/context.txt`
4. grep 道统关键词库（明阳/离火/府水/坎水/牝水/庚金/库金/艮土/巽木/太阴/太阳/少阳/厥阴/紫炁/真炁/青宣/邃炁/寒炁/晞炁/煞炁/霄雷/玄雷/元雷/全丹/剑道/玉真/瑞炁/清炁/密汎/衡祝/并火/牡火/真火 等）
5. 输出 CSV：`页面名,目录,品阶,出现次数,关键词命中,证据句子`

**人工审阅**：
- 对关键词命中的页面快速确认是否为真正证据
- 区分 A/B/C/D 四类

**预计耗时**：脚本 10 min + 人工审阅 30 min

---

### 阶段 1：A 类批量修正（人工 + 脚本，1 轮）

**范围**：预筛选确认的 A 类页面
**方式**：
1. 人工逐页确认证据可靠性
2. 按 v3.1 S3 格式批量写入：`| 道统 || [[道统-XX|XX]] ✅` + 置信度说明
3. 执行三同步校验（info 表 / 首句 / Category）

**预计数量**：15-25 个
**预计耗时**：30 min

### 阶段 3：C 类批量标注（脚本，1 轮）


**预计数量**：60-80 个

## 四、风险与预案

| 风险 | 预案 |
|------|------|
| 预筛选脚本误分类（将有证据的标为C类） | C类批量标注前人工抽检 20%，发现误分类即时移入B类 |
| Agent 推断过度（把⚡标成✅） | 每批抽查 10%，违反 v3.1 铁律3 的退回修正 |
| Context 文件缺失 | 对缺失 context 的页面回源 grep 原著 txt |
| 天然无道统的界定争议 | 以「是否人工炼制」为界：人工炼制→应有道统；天然生成→可无道统 |
