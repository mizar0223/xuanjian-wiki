# 基本信息（品阶/道统）工作规范 v3

> **制定背景**：六雷玄罚令品阶错误（符箓→筑基法器）+ 道统错误（霄雷→玄雷）、天一淳元道统错误（水德/太阴→牝水）、**望晋玄衍丹道统错误（全丹→瑞炁）**复盘
> **核心教训**：AI 推理冲动压过原文权威证据，是基本信息错误的共性根因；**宽窗 context 未被优先使用，导致关键直接证据（如章名《瑞成》、青衍「瑞炁之丹」）被遗漏**
> **适用范围**：所有 wiki 页面 `== 基本信息 ==` 区块的品阶/道统字段写入与修正
> **所属工作流**：本文是 `WORKFLOW.md` §2「品阶三同步铁律」的专项实施细则。上传发布见 `WORKFLOW.md` §3，多 Agent 协作见 `WORKFLOW.md` §4。

## 一、核心原则：原文优先，推理从属

**原文直接标注是一级权威**，所有推理必须从属于原文，不得覆盖原文。

### 判定优先级链

```
1. 原文直接标注 → ✅ 直接写入
2. 所属人物道统推断 → ⚡ 必须附推理依据
3. 功能/属性推断 → ⚡ 必须附推理依据+类比参照
4. 名字关键词推断 → ⚡⚠️ 最低优先级，必须附警告
5. 无法判断 → ❓ 不猜测
```

## 二、三大禁止（铁律）

### 禁止1：名面联想覆盖原文

**当名字中的字面特征与原文直接标注冲突时，以原文为准，字面联想必须让步。**

触发条件：实体名中含某字 → AI 基于该字做品类/道统推断 → 但原文有更明确的标注。

| 实体 | 名面联想 | 原文证据 | 正确值 |
|---|---|---|---|
| 六雷玄罚令 | 「令」→符箓 | 「乃是筑基法器中少见的成套的法器」（第583章） | 筑基法器 ✅ |
| （潜在） | 「火」→离火 | 可能是明阳/并火/牡火 | 需原文验证 |
| （潜在） | 「金」→庚金 | 可能是库金 | 需原文验证 |
| （潜在） | 「水」→府水 | 可能是牝水/坎水/渌水 | 需原文验证 |

**判定规则**：凡 AI 基于名字做推断，必须先 grep 原文验证。原文有直接标注的，以原文为准。

### 禁止2：上位泛化替代精确指定

**当原文指定了具体道统名，不得用上位概念替代。**

触发条件：AI 认为"原文指定太窄"，试图上推到更广泛的分类。

| 实体 | 原文指定 | 错误上位泛化 | 正确值 |
|---|---|---|---|
| 天一淳元 | 「【天一淳元】是牝水」 | 水德/太阴（牝水，非单一限定） | 牝水 ✅ |
| 六雷玄罚令 | 「却是『玄雷』」 | 霄雷（因持有者修霄雷） | 玄雷 ✅ |

**判定规则**：
- 道统归属看**法器/灵物自身**的道统属性，而非持有者的道统
- "水德""金德""火德"不是标准道统名——标准道统名是牝水/府水/渌水/坎水/庚金/库金/明阳/离火等具体名
- 持有者道统 ≠ 法器道统（如六雷玄罚令：持有者修霄雷，法器属玄雷）

### 禁止3：推理结果标✅

**✅ 只在原文有直接文字证据时使用。凡涉及推理，一律标⚡或⚡⚠️。**

| 标注 | 条件 | 示例 |
|---|---|---|
| ✅ | 原文明确写出品阶/道统名 | 「乃是筑基法器」→品阶✅；「是牝水」→道统✅ |
| ⚡ | 基于间接证据推理 | 「稳压当今灵器」→灵宝⚡；持有者修明阳→明阳⚡ |
| ⚡⚠️ | 仅基于名字推断（最低信心） | 「令」→符箓⚡⚠️；「金阳」→明阳⚡⚠️ |
| ❓ | 无法判断 | 不猜测，标注「待确认」 |

**置信度标注必须与证据等级严格对应，不可"拔高"。**

## 三、3步强制写入流程（不可跳步）

> **前提**：宽窗 context.txt 是原始一手资料（从39个原著txt提取的500+500字窗口），wiki 页面的【原文引用】是二手挑选资料（仅包含我挑选的2-3条引用）。
> 因此流程重心在"从 context 中全面提取证据"——context 包含章名、人物对话、旁白等完整信息，远比我挑选的引用更全面。

### S1 证据提取（从 context 与 wiki 中互补提取）

**核心原则：宽窗 context 是一手原始资料，wiki 页面的【原文引用】是二手挑选资料。优先从 context 提取，wiki 引用仅作补充验证。**

#### §1.1 读取优先级（不可颠倒）

```
第一步：通读宽窗 context.txt（强制）
  → 逐窗口阅读，标记所有含品阶/道统/属性关键词的句子
  → 特别注意章名（如第944章《瑞成》——章名本身就是证据）
  → 特别注意人物对话中的定性语句（如青衍称「瑞炁之丹」）

第二步：对照 wiki 页面的【原文引用】区（补充）
  → wiki 有而 context 无的引用 → 补充进证据池
  → wiki 无而 context 有的关键句 → 优先采用（wiki 引用是我挑选的，可能有遗漏）
  → 两边冲突时 → 以 context 为准（context 是完整宽窗，wiki 引用是裁剪后的子集）

第三步：仅在上述两处均无证据时，才回源 grep 原著全文
```

**禁止行为**：
- ❌ 跳过 context，直接读 wiki 引用区就开始推理
- ❌ 把 wiki 页面已填的字段值（如道统=全丹）当成既定事实，反向去 context 找支撑
- ❌ 忽略章名中的关键词（章名是作者给出的最精炼概括）

#### §1.2 提取方法

从 context.txt 中提取所有与品阶/道统直接相关的原文证据：

| 证据类型 | 示例 | 优先级 |
|----------|------|--------|
| 章名关键词 | 第944章《**瑞**成》 | 高（作者概括） |
| 人物直接定性 | 青衍：「『瑞炁』之丹」；李曦明：「好一枚**瑞丹**」 | 最高（✅级） |
| 旁白直接标注 | 「乃是筑基法器中少见的成套的法器」 | 最高（✅级） |
| 材料属性标注 | 「紫府『瑞炁』灵物炼成」 | 高 |
| 使用者道统 | 「本王修行『瑞炁』」 | 中（需区分使用者≠造物本身） |
| 功能效果描述 | 「炼成了望晋**瑞**体」 | 中（语义关联，需验证） |

#### §1.3 提取示例（望晋玄衍丹——正确流程）

#### §2.4 权威文档对推理的约束

**硬约束（违反即错误）**：
1. **道统名录以神通仙基汇总为准**：凡不在55+道统名录中的名称，不得作为道统字段值写入（如"水德""金德""火德""木德""土德"均为上位分类，非道统名）
2. **品阶定义以原著体系梳理为准**：品阶术语的精确定义（如"灵物=性命极其丰沛齐全""灵资=性命大都齐全"）不可自行发挥
3. **判定规则以分类判定规则为准**：优先级链和关联词映射为标准判定依据，不可跳过

**软约束（参考，可注明偏差）**：
1. **金性定义**：神通仙基汇总中每个道统的金性描述（如牝水「胎息大臟玄牝性」），可用于辅助验证道统归属的合理性
2. **下位/替参关系**：神通仙基汇总中「→」标记的下位/替参神通，暗示道统间的亲缘关系，可辅助判断但不可替代原文
3. **道轨归属**：原著体系梳理中的道轨-道统映射（如某道统属某道轨），提供层级上下文，但道统字段只写最具体层

## 四、特殊场景处理

### 场景1：法器道统 ≠ 持有者道统

**规则：法器/灵物自身的道统归属优先于持有者道统。**

当原文明确说法器与持有者"不契合"时，法器的道统是法器自身的属性，不是持有者的道统。

示例：六雷玄罚令
- 法器道统：玄雷（原文「却是『玄雷』」）
- 持有者道统：霄雷（李清虹修霄雷）
- 正确标注：道统=玄雷 ✅，在「所属道统」区说明与持有者的不契合关系

### 场景2：道统名在上位分类中的归属

**规则：不在基本信息中写上位分类名。**

道统字段只写具体道统名（牝水/府水/庚金/明阳等），不写上位分类（水德/金德/火德等）。上位分类关系可在「所属道统」区的正文中补充说明。

### 场景3：多道统归属

**规则：原文明确属多道统时，主字段写最直接的道统，其余在「所属道统」区说明。**

如原文说某灵物"兼具XX与YY之性"，主字段写最优先的道统（原文强调的那个），附 ⚡ 标注，在「所属道统」区详细说明。
