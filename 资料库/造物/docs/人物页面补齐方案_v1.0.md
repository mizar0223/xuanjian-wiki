# 人物页面补齐方案 v1.0

> **制定背景**：人物维度 19 页已建但普遍为骨架级（1.8K–5.4K），缺生平章节标注、名字由来、法宝/神通详细列表、关系网络。需按 workflow 3.1 证据提取规范执行补齐。
> **适用范围**：`pages/20-人物/` 目录下全部人物 wiki 页面的深度补全
> **所属工作流**：本文是 `WORKFLOW.md` §5「Rescue 批次模板」的专项实施细则。品阶/道统判定见 `docs/workflow-基本信息更新工作规范.md`（v3.1）

---

## 一、核心原则

### 1.1 人物页 ≠ 造物页

人物页面的核心差异：

| 维度 | 造物页 | 人物页 |
|------|--------|--------|
| **时间线** | 无（静态物品） | 有（生平分阶段） |
| **关系网络** | 持有者/炼制者 | 家族+师承+盟友+仇敌 |
| **性格分析** | 无 | 从对话/行为提炼 |
| **原文引用** | 功能描述为主 | 对话+旁白+事件 |
| **章节标注** | 可选 | **必须**（生平每阶段） |

### 1.2 原文优先，章节标注为硬要求

人物生平中**每个关键事件必须标注原文章节**（如「第 583 章《玄罚》」）。这是人物页与造物页最核心的区别，也是当前 19 页普遍缺失的要素。

### 1.3 宽窗 context 优先（workflow 3.1 S1）

人物生平的时间线梳理**必须**基于宽窗 context 中的原文证据，禁止仅凭百科全书概括或印象填充。

## 二、深度标准（⭐⭐⭐ / ⭐⭐ / ⭐）

### ⭐⭐⭐ 深度（目标标准）

| 模块 | 达标要求 | 证据等级 |
|------|----------|----------|
| **名字由来** | 本名逐字拆解 + 道号含义 + 命名典故 | ⚡ 推断（附依据） |
| **生平** | 按时间线分 ≥3 阶段，每阶段有关键事件 + **原文章节标注** | ✅ 原文直接 |
| **性格** | 从具体对话/行为提炼，附原文依据 | ✅ 原文直接 |
| **修炼体系** | 仙基/箓气/道统详细说明 + 突破历程（含章节） | ✅ 原文直接 |
| **持有法宝** | 详细列表 + 获取来源 + 使用场景 | ✅/⚡ |
| **修炼神通** | 列表 + 品阶 + 修炼章节 | ✅/⚡ |
| **重要事件** | ≥5 个，带章节标注 + 事件影响 | ✅ 原文直接 |
| **关系网络** | 家族 + 师承 + 盟友 + 仇敌 ≥8 个节点 | ✅/⚡ |
| **原文引用** | ≥5 条（百科全书 + 原著正文对话/旁白） | ✅ 原文直接 |

### ⭐⭐ 深度（可接受标准）

- 生平：2–3 个阶段，**部分有章节标注**
- 性格：有描述，未附原文依据
- 法宝/神通：有列表，获取来源不详
- 重要事件：3–5 个
- 关系网络：≥5 个节点
- 原文引用：3–5 条

### ⭐ 深度（当前多数页面）

- 生平：1–2 段概括，**无章节标注**
- 性格：标签式描述
- 法宝/神通：缺失或极简略
- 原文引用：1–2 条百科全书式引用

## 四、4 阶段执行流程

### Phase 0 · 提取人物宽窗 context（30–60 min）

**前提**：`context/` 下无 `20-人物/` 目录，需先为 16 人提取。

**步骤**：

```
1. 复用 batch_extractor.py 的 extract_one_entity() 核心函数
2. 传入 16 个人物全名 + 品类 "20-人物"
3. 输出目录：.workbuddy/wide_window/context/20-人物/{人物名}/
   - context.txt（宽窗文本）
   - meta.json（提及次数、窗口数、字数统计）
```

**风险预警**：
- 人物贯穿全书，提及次数可能 200–500 次/人
- 宽窗 context 总量预计 50–100M，提取时间比造物维度长
- **应对**：提取后先读 meta.json，若合并窗口 >50 个，优先读首尾窗口 + 关键章节窗口

## 五、单页工作流（按 workflow 3.1 S1→S2→S3）

### S1 证据提取（从 context 中全面提取）

**§1.1 读取优先级（不可颠倒）**

```
第一步：通读人物宽窗 context.txt（强制）
  → 逐窗口阅读，标记所有含人物关键事件的句子
  → 特别注意章节标题（如第 583 章《玄罚》——章名暗示事件性质）
  → 特别注意人物对话（性格/决策/关系的一手证据）

第二步：对照 wiki 页面的【原文引用】区（补充）
  → wiki 有而 context 无的引用 → 补充进证据池
  → wiki 无而 context 有的关键句 → 优先采用
  → 两边冲突时 → 以 context 为准

第三步：仅在上述两处均无证据时，才回源 grep 原著全文
```

**§1.2 提取方法**

从 context.txt 中提取所有与人物直接相关的原文证据：

| 证据类型 | 示例 | 优先级 | 用途 |
|----------|------|--------|------|
| 章名关键词 | 第 583 章《玄罚》 | 高 | 生平阶段划分 |
| 人物对话 | 李周巍喝死管坎 | 最高 | 性格/事件 |
| 旁白直接标注 | 「乃是筑基前期【浩瀚海】」 | 最高 | 修炼体系 |
| 法宝获取 | 「李曦明接过【重明洞玄屏】」 | 高 | 持有法宝 |
| 关系定性 | 「师萧元思」 | 高 | 关系网络 |
| 事件结果 | 「仙基崩碎而亡」 | 高 | 重要事件 |

**§1.3 禁止行为**

- ❌ 跳过 context，直接基于百科全书概括开始写作
- ❌ 把 wiki 页面已填的生平段落当成既定事实，反向去 context 找支撑
- ❌ 生平段落不写章节标注（这是人物页的核心缺陷）
- ❌ 用标签式性格描述替代原文对话证据

### S2 内容组织（按人物页结构重组证据）

将 S1 提取的证据按以下结构重组：

```
1. 名字由来（本名 + 道号）
2. 生平（按时间线分阶段，每阶段含：事件概述 + 原文章节标注 + 影响）
3. 性格（从对话/行为提炼，附原文依据）
4. 修炼体系（仙基 → 箓气 → 道统，含突破章节）
5. 持有法宝（列表 + 获取来源 + 使用场景）
6. 修炼神通（列表 + 品阶 + 修炼章节）
7. 重要事件（≥5 个，时间线表格）
8. 关系网络（家族 + 师承 + 盟友 + 仇敌）
9. 原文引用（≥5 条）
```

### S3 写入 + 三同步校验

**写入后立即执行**：

```
□ info 表人物字段 = 首句人物 = Category 人物标签
□ 首句格式：'''人物名'''（道号'''XX'''），是《玄鉴仙族》中[[势力-XX|XX]]的...
□ 生平每阶段含原文章节标注（第 NNN 章《XX》）
□ 原文引用 ≥5 条
□ Category：[[Category:人物]] [[Category:李家/外族势力]] [[Category:道统-XX]]
```

## 七、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 人物提及次数过多，context 量爆炸 | 提取慢、阅读难 | 提取后先读 meta.json；若窗口 >50，优先读首尾 + 关键章节 |
| 人物有多名/道号/尊称 | context 提取遗漏 | 核心名提取为主，道号补充提取为辅 |
| 人物关系网络复杂 | 写作耗时超预期 | 先保证生平 + 法宝 + 事件达标，关系网络可降至 ⭐⭐ 标准 |
| 与现有造物页内容重复 | 持有法宝等模块与造物页重叠 | 人物页侧重「获取来源 + 使用场景」，造物页侧重「功能威能」，互补而非重复 |
