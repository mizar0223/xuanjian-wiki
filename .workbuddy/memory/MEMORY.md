## 分类体系规则（2026-05-12 确立）

### 三维分类模型
| 维度 | 含义 | 示例 |
|---|---|---|
| **类型** | 造物本质形态 | 灵气、灵物、法器、灵器、灵宝、法宝、符箓、丹药 |
| **层级** | 修为门槛 | 凡人/胎息/练气 → 筑基 → 紫府 → 金丹/真君 → 仙人 |
| **品质** | 同类型同级品质 | 下品、中品、上品、极品、古 |

例：'''上品紫府灵物''' = 类型(灵物) + 层级(紫府) + 品质(上品)

### 品类层级
灵物(自然物) < 法器 < 灵器 < 灵宝(造物) | 功法/法门/法术/丹方(术法)

### "书"类条目判定规则
- **器物本质**（归灵宝）：原文用作收储、索引、钥匙、手持战斗→崆峒仙书、手持仙书、七十二洞玄书、万训科律书、太栩紫炁书
- **道统本质**（归功法/法门）：原文定义为道统传承、记载神通法门→帝官有誓书、金书、青玄谶书、太昱孚性书
- 关键判据：搜索原文看实际使用方式，而非仅凭名称推断

### 权威文档（基本信息工作规范.md v3.1 引用机制）
1. 分类判定规则.md — 分类决策逻辑

### 关键原著设定
- **古灵器 = 灵宝**：长奚真人原话"其实就是天变以前的古灵器……如今却要把他叫灵宝了"
- **灵胚 ≠ 独立类型**：是紫府法器（灵器）的雏形/过渡阶段
- **衡祝** = 【并古法】道统属性，非灵宝子类
- **紫府灵火/灵水**：层级+形态属性统称，非独立类型
- **中品法器** = 练气级，紫府修士兵器称"灵器"不称"法器"

## 铁律与操作规范

### 品阶三同步（P0）
每个实体必须 info表品阶 / 首句"是…的[[X]]" / Category **三处一致**。修改品阶时务必三处同步。

### 三禁止（基本信息工作规范 v3.1）
1. **禁止名面联想覆盖原文**：六雷玄罚令 ≠ 符箓（"令"字联想），= 筑基法器（原文）
2. **禁止上位泛化替代精确指定**：天一淳元道统 ≠ 水德/太阴，= 牝水（原文）
3. **禁止推理标✅**：仅原文直接证据标✅，推断标⚡，无法判断标❓

### 数据可信度标记
- ✅ 明确（原文证据）
- ⚡ 推断（间接证据）
- ⚡⚠️ 名面推断（最低信心）
- ❓ 未知（不猜测）

### 持有者道统陷阱
**法器道统 ≠ 持有者道统**，原文有直接定性时必须以原文为准。
- 都道铃：持有者都仙 ≠ 法器清炁（原文"是『清炁』"）
- 六雷玄罚令：持有者霄雷 ≠ 法器玄雷（原文"却是『玄雷』"）

### .wiki文件迁移操作
迁移条目到新分类目录时，必须修改3处：
1. 开篇定义行（如 `[[灵宝]]` → `[[功法]]`）
2. 品阶字段（同上）
3. Category标签（如 `[[Category:灵宝]]` → `[[Category:功法]]`）
用 `mv` 移动文件后，必须先 Read 新路径的文件，才能用 Edit 修改内容。

### 内链规范（v2，2026-05-12 更新）
`[[XX]] [[人物-XX]] [[势力-XX]] [[地点-XX]] [[道统-XX]] [[神通-XX]] [[事件-XX]]`
- 造物条目已统一无前缀：`[[太阴月华]]` 而非 `[[造物-太阴月华]]`
- 人物仍用 `[[人物-XX]]` 前缀
- 线上 `造物-XX` → `XX` 重定向保留，旧链不破

## 上传基础设施

```bash
cd /Users/leoshi/AIBOOK/xuanjian/wiki
set -a && source .env && set +a
python3 scripts/upload_pages.py --only "资料库/造物/pages" --summary "xxx"
```

| 参数 | 作用 |
|---|---|
| `--prefix-map` | 目录→前缀映射（内置 DEFAULT_PREFIX_MAP 已含造物子目录→空前缀） |
| `--no-prefix` | 强制无前缀（特殊场景） |
| `--force` | 覆盖冲突 |
| `--dry-run` | 预览不写入 |

凭证：`.env` 中 WIKI_USER / WIKI_PASS

## Lessons Learned（重要教训）

1. **宽窗context优先**（v3.1铁律）：context是一手原始资料（500+500字），wiki引用区是二手挑选资料（仅2-3条）。顺序颠倒导致望晋玄衍丹道统错误。
2. **持有者道统 ≠ 法器道统**：都道铃/六雷玄罚令两次踩坑，原文直接定性优先。
3. **名字联想陷阱**：都道铃≠都仙（名含"都道"实为清炁）；六雷玄罚令≠符箓（"令"≠符箓）
4. **章名是证据**：第944章《瑞成》的"瑞"是瑞炁道统线索，不可忽略。
5. **batch_generator 会覆盖agent输出** → MUST先检查文件大小
6. **文件名≠页面标题**：L前缀文件名不能直接当stem用
7. **00-体系专题页用 `--no-prefix`**：否则创建`造物-灵宝`错误页面
8. **考据文档正文引用实体会被遗漏**：仅标题级实体进入提取流程，正文【XX】引用的被静默丢弃
9. **3 Agent是硬上限**：超过会僵尸+context膨胀+决策退化
10. **品阶并列推断要谨慎**：大夏郢铜甲证据不足曾误推，已回退
11. **上传前缀必须一致**：两轮上传混用前缀导致454对重复页面，已清理。同一维度条目前缀策略不可变动
12. **线上 move 保留重定向**：`action=move` 默认创建重定向（`noredirect` 不传），安全无破链
